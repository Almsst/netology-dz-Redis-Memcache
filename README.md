# Домашнее задание: Кеширование (Memcached / Redis)


## Задание 1. Проблемы, решаемые кешированием

Кеширование помогает решить следующие типовые проблемы:

1. Высокая нагрузка на базу данных – при частых одинаковых запросах (например, чтение популярных товаров в интернет-магазине) кеш берёт на себя часть чтения, снижая число обращений к БД.
2. Медленные ответы API – сторонние сервисы или собственные тяжёлые вычисления могут выполняться долго. Кеширование результата (например, на 1 минуту) ускоряет повторные запросы.
3. Пиковые нагрузки (traffic spikes) – без кеша серверы и БД могут не справиться с внезапным наплывом пользователей. Кеш (даже небольшой) «сглаживает» пики за счёт отдачи горячих данных из памяти.
4. Большие вычислительные затраты – агрегация отчётов, рендеринг сложных страниц, пересчёт рекомендаций. Кеширование промежуточных или финальных результатов экономит CPU.
5. Высокая задержка при работе с медленными хранилищами – дисковая БД или удалённый файловый сервер. in‑memory кеш (Redis, Memcached) даёт доступ к данным за микросекунды.
6. Проблемы совместного использования состояния – например, хранение сессий пользователей в веб‑кластере без привязки к конкретному серверу (централизованный кеш сессий).
7. «Горячие» ключи в БД – если одна запись читается тысячами запросов в секунду, кеш полностью разгружает БД по этому ключу.
8. Лишние сетевые вызовы – при микросервисной архитектуре часто одни и те же данные (информация о пользователе, настройки) запрашиваются разными сервисами. Кеш (общий или локальный) сокращает число дублирующихся вызовов.
Использование Redis или Memcached позволяет решить эти проблемы за счёт быстрой памяти, поддержки TTL (время жизни кеша), атомарных операций, репликации и кластеризации.
